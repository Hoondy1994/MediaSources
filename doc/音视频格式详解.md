# 音视频格式详解

## 一、音频格式

### 1. 无损格式（Lossless）

| 格式 | 全称 | 特点 |
|------|------|------|
| **WAV** | Waveform Audio File Format | 未压缩，音质最佳，体积大，常用于专业录音 |
| **FLAC** | Free Lossless Audio Codec | 无损压缩，体积约为 WAV 的 50–60%，开源免费 |
| **ALAC** | Apple Lossless Audio Codec | Apple 无损格式，常用于 iTunes、iOS |
| **APE** | Monkey's Audio | 压缩率高，解码较耗 CPU |
| **AIFF** | Audio Interchange File Format | Apple 未压缩格式，常用于 Mac 音频制作 |

### 2. 有损格式（Lossy）

| 格式 | 全称 | 特点 |
|------|------|------|
| **MP3** | MPEG-1 Audio Layer 3 | 最普及，128–320 kbps，兼容性最好 |
| **AAC** | Advanced Audio Coding | 同码率下音质优于 MP3，流媒体常用 |
| **OGG Vorbis** | Ogg Vorbis | 开源，音质好，支持多声道 |
| **WMA** | Windows Media Audio | 微软格式，在 Windows 上较常见 |
| **Opus** | Opus | 低延迟，适合实时通话、流媒体，WebRTC 常用 |

### 3. 专业/工程格式

| 格式 | 说明 |
|------|------|
| **MIDI** | 存储音符和指令，不存储波形，文件很小 |
| **DSD** | Direct Stream Digital，SACD 使用，采样率高 |
| **PCM** | 脉冲编码调制，数字音频的原始形式 |

---

## 二、视频格式（容器）

### 1. 常见容器格式

| 格式 | 全称 | 特点 |
|------|------|------|
| **MP4** | MPEG-4 Part 14 | 最常用，兼容性好，支持 H.264、H.265、AAC 等 |
| **MKV** | Matroska Video | 支持多音轨、多字幕，适合高清和蓝光 |
| **AVI** | Audio Video Interleaved | 老格式，兼容性好，压缩效率一般 |
| **MOV** | QuickTime Movie | Apple 格式，常用于专业剪辑 |
| **WebM** | Web Media | 面向 Web，开源，常用于 VP8/VP9 |
| **FLV** | Flash Video | 早期流媒体常用，现已较少 |
| **WMV** | Windows Media Video | 微软格式 |
| **TS/M2TS** | MPEG Transport Stream | 常用于直播、蓝光、数字电视 |
| **3GP** | 3GPP | 面向手机，体积小 |
| **MPEG** | MPEG-1/2 | 老标准，VCD/DVD 常用 |

### 2. 视频编码（Codec）

| 编码 | 全称 | 特点 |
|------|------|------|
| **H.264/AVC** | Advanced Video Coding | 最普及，兼容性最好，蓝光、流媒体常用 |
| **H.265/HEVC** | High Efficiency Video Coding | 同画质下码率约为 H.264 一半，需授权 |
| **VP9** | Google VP9 | 开源，YouTube 常用，无专利费 |
| **AV1** | AOMedia Video 1 | 新一代开源编码，压缩率更高，解码较耗资源 |
| **VP8** | Google VP8 | WebRTC 常用，较老 |
| **MPEG-2** | MPEG-2 Video | DVD、数字电视常用 |
| **MPEG-4 Part 2** | DivX/Xvid | 早期网络视频常用 |
| **ProRes** | Apple ProRes | 专业剪辑用，质量高、体积大 |

### 3. 音频编码（在视频中）

| 编码 | 说明 |
|------|------|
| **AAC** | 视频中最常用 |
| **AC3/Dolby Digital** | 5.1 声道，蓝光、影院常用 |
| **DTS** | 多声道，蓝光常用 |
| **PCM** | 未压缩，音质最好 |
| **Opus** | WebM 常用，适合 Web 和实时通信 |

---

## 三、关键概念

### 1. 容器 vs 编码

- **容器**：如 MP4、MKV，决定文件结构、多轨、元数据等。
- **编码**：如 H.264、AAC，决定如何压缩音视频数据。
- 同一容器可搭配不同编码，例如：MP4 可包含 H.264 + AAC 或 H.265 + AAC。

### 2. 常见组合

| 用途 | 推荐组合 |
|------|----------|
| 通用兼容 | MP4 + H.264 + AAC |
| 高压缩比 | MKV + H.265 + AAC |
| Web 播放 | WebM + VP9 + Opus 或 MP4 + H.264 + AAC |
| 实时通话 | WebRTC（VP8/VP9/H.264 + Opus） |
| 专业剪辑 | MOV + ProRes + PCM |

### 3. 分辨率与帧率

- **分辨率**：720p、1080p、4K、8K 等。
- **帧率**：24fps（电影）、30fps、60fps（游戏、体育）等。
- **码率**：单位时间数据量，影响画质和文件大小。

## 四、Linphone 相关

在 Linphone 这类 VoIP/视频通话项目中，常见使用：

- **视频**：H.264、H.265、VP8、VP9
- **音频**：Opus、G.711、G.722、AAC 等
